# IP Boundary

## Public / Client-Facing Layer

This repository may be used to present:

- product category
- commercial offer
- diagnostic deliverables
- high-level methodology
- sample schema structures
- starter implementation scaffold
- pilot plan

## Protected Elyria Layer

This repository does not disclose the full private estate, including but not limited to:

- private artifact ledgers
- internal verifier surfaces
- authority-revocation machinery
- protected receipt implementations
- full runtime admission kernel
- proprietary consequence math
- private artifact/receipt/verifier inventory
- client-specific findings

## Engagement Boundary

Client work should be scoped so that the client receives the contracted diagnostic, report, pilot deliverables, and agreed implementation artifacts without acquiring rights to pre-existing Elyria systems, methods, proof surfaces, naming, architecture, or runtime IP unless separately licensed in writing.
